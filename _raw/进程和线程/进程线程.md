### 进程线程

#### 进程和线程的区别

* 前置条件：CPU太快了，所以会调度执行各个命令集。

|     |     |     |
| :--- | ---: | :---: |
| Col1 | 进程  | 线程  |
| 基本概念 | 资源分配、调度的基本单位 | CPU调度的基本单位 |
| 拥有资源 | 独立的地址空间 | 各自的寄存器和堆栈、共用data、bss |
| 调度  | 实际上调度的是进程的主线程 | 调度的基本单位 |
| 调度代价 | 上下文切换 | 较小  |
| 通信  | 进程间通信、socket | 进程内直接通信、socket |
| 数据交互 | 共享内存、程序到内核到程序 | 进程内直接拷贝 |
| 稳定性 | 进程崩溃不会影响其他进程\[1\] | 一个线程崩溃（整个进程崩溃） |

注：1. 进程地址空间边界
