### inode

文件存储在硬盘上，硬盘的最小存储单位叫做“扇区”（Sector）。每个扇区储存512字节（相当于0.5KB）。
操作系统读取硬盘的时候，不会一个个扇区的读取，这样效率太低，而是一次性连续读取多个扇区，即一次性读取一个“块”（block）。这种由多个扇区组成的“块”，是文件存取的最小单位。
“块”的大小，最常见的是4KB，即连续八个sector组成一个block。
文件数据都储存在“块”中，那么很显然，我们还必须找到一个地方储存文件的“元信息”，比如文件的创建者、文件的创建日期、文件的大小等等。这种储存文件元信息的区域就叫做inode，中文译名为”索引节点“。

#### inode包含的信息

* Size 文件的字节数
* Uid 文件拥有者的User ID，Gid 文件的Group ID
* Access 文件的读、写、执行权限
* 文件的时间戳，共有三个：
	* Change 指inode上一次变动的时间
	* Modify 指文件内容上一次变动的时间
	* Access 指文件上一次打开的时间
* Links 链接数，即有多少文件名指向这个inode
* Inode 文件数据block的位置
* Blocks 块数
* IO Blocks 块大小
* Device 设备号码

可以使用stat 命令来查看inode信息：
```
  File: ‘consumer.go’
  Size: 1000          Blocks: 8          IO Block: 4096   regular file
Device: fe01h/65025d    Inode: 656596      Links: 1
Access: (0644/-rw-r--r--)  Uid: ( 1000/     zzy)   Gid: ( 1000/     zzy)
Access: 2018-12-01 15:08:02.824702772 +0800
Modify: 2018-12-01 15:07:01.253615113 +0800
Change: 2018-12-01 15:07:58.704763818 +0800


```

#### inode速记

* 每个inode节点的大小，一般是128字节或256字节。每个文件(linux一切皆文件)都有一个inode。所以小文件太多，会有问题( inode号并不是无限的，如果一个分区的节点数被使用完了，那么即使磁盘空间还有剩余也不能再存放任何数据)
* inode是文件的唯一标识，而不是文件描述符或者文件名；
* 用户通过文件名，打开文件。实际上，系统内部这个过程分成三步：首先，系统找到这个文件名对应的inode号码；其次，通过inode号码，获取inode信息；最后，根据inode信息，找到文件数据所在的block，读出数据。
* 删除一个文件名，就会使得inode节点中的”链接数”减1。当这个值减到0，表明没有文件名指向这个inode，系统就会回收这个inode号码，以及其所对应block区域。硬连接会增加links值。
* cp file1 file2 如果file2存在，不会修改file2的inode；相当于打开file2，将file1的内容塞入file2，因为如果file2被映射到内存，直接修改会导致进程崩溃。最好先rm，再mv file1 file2
* mv file1 file2只是文件改名，inode一样，Change时间改变，Access和Modify时间不变。
* 文件被应用程序打开了，使用losf -p pid，可以查看pid号进程打开的文件和对应的inode，删除该inode对应的文件，不会对应用程序造成影响，进程结束时才会实际删除被删除的文件。

#### 硬连接和软连接

* 硬连接：共享inode节点，只是增加inode计数。缺点是不能跨文件系统
* 软连接：有自己的inode，只是使用路径名作为指针。可以跨文件系统
